# manus量化策略波动率加权版-每周五操作版 (GitHub Actions 版)

本项目是一个基于 Python 的全自动量化交易策略信号系统。它脱离了传统的聚宽（JoinQuant）等量化平台，通过 **GitHub Actions** 实现完全免费的云端定时运行，并利用 **PushPlus** 将调仓指令实时推送到微信。

## 🧠 策略核心逻辑

本策略采用 **动量轮动 + 宏观择时 + 严格风控** 的混合架构，以周频（每周五尾盘）运行。

### 1. 核心资产池
* **市场情绪锚定 (Market Anchor):** 沪深300 ETF (`510300`)
* **进攻池 (Alpha Pool):** 新能源汽车 (`512480`)、创业板50 (`159206`)、电池ETF (`159755`)、医疗ETF (`159819`)
* **防守池 (Beta Pool):** 红利低波 (`512890`)、黄金ETF (`518880`)、纳指ETF (`513100`)
* **现金底仓 (Cash):** 银华日利 (`511880`)

### 2. 择时与轮动引擎
* **RSRS 择时:** 以沪深300为锚，计算过去250天内，18日高低价的线性回归斜率（Beta）的标准化得分。当 RSRS 得分 > 0.8 时，视为市场处于强势上行通道。
* **进攻模式:** 当 RSRS > 0.8 时，在进攻池中筛选近11日涨幅 > 3% 的标的，选取涨幅最高的一只。
* **防守模式:** 当 RSRS 不满足进攻条件时，切换至防守池，计算各标的近21日的“收益率/波动率”比值，选取表现最稳健的一只。
* **空仓防守:** 若进攻或防守池均无符合条件的标的，则全仓买入银华日利（现金池）吃利息。
* **权重分配:** 选中标的后，采用**21日波动率倒数加权**算法计算资金分配比例，实现风险平价。

### 3. 硬止损与熔断机制 (Risk Control)
* **系统级回撤保护:** 每次运行会记录账户的“历史最高峰值净值”。
* **熔断触发:** 如果当前净值较历史峰值回撤达到或超过 **15%**，系统将触发硬止损。
* **绝对冷静期:** 触发止损后，系统会自动清空所有持仓（转为现金），并强制进入 **2周的冷静期**。冷静期内忽略一切买入信号，防止在极端行情中来回磨损。
